WHAT IS THIS?
-------------
Slogr Agent is a network measurement agent that runs on cloud VMs and measures
network quality between endpoints using TWAMP (Two-Way Active Measurement
Protocol, RFC 5357). It reports RTT, jitter, packet loss, and SLA grade for
27+ application profiles (VoIP, gaming, video conferencing, etc.).

R2 is the production-ready Kotlin/JVM rewrite. It replaces the legacy Python/C++
agent and adds API key authentication, OTLP metrics export, state-aware startup,
and credential management.

WHERE TO GET IT
---------------
Repository:  https://github.com/nasimkz/slogr-agent-kotlin
Branch:      master (R2 has been merged into master)
CI Status:   Both master and R2 branches are CI-green

Artifacts built from master:
  - Shadow JAR:     app/build/libs/slogr-agent-all.jar
  - Docker image:   built via:  docker build -f Dockerfile -t slogr-agent .
  - Native library: native/build/libslogr-native.so  (Linux glibc)
                    native/build/libslogr-native-musl.so  (Linux musl/Alpine)


================================================================================
  AGENT STATES
================================================================================

The agent operates in one of three modes determined by SLOGR_API_KEY:

  ANONYMOUS   No API key set, or invalid format.
              - stdout logging only
              - No OTLP export, no RabbitMQ/Pub/Sub
              - Useful for testing TWAMP reflector without a cloud account

  REGISTERED  API key starts with sk_free_
              - OTLP metrics export enabled
              - stdout logging
              - No RabbitMQ/Pub/Sub integration

  CONNECTED   API key starts with sk_live_
              - Full cloud integration: OTLP + RabbitMQ + Pub/Sub
              - Receives measurement schedules from backend
              - Reports results back to Slogr cloud

---  OPTION 1: DOCKER (Recommended for production)  ---

Requirements:
  - Docker
  - Linux host (for CAP_NET_RAW / TWAMP raw sockets)

Build the image:
  docker build -f Dockerfile -t slogr-agent:latest .

Run (ANONYMOUS — reflector-only, no cloud):
  docker run -d \
    --name slogr-agent \
    --cap-add NET_RAW \
    --cap-add NET_BIND_SERVICE \
    slogr-agent:latest daemon

Run (REGISTERED — with sk_free_ key for OTLP):
  docker run -d \
    --name slogr-agent \
    --cap-add NET_RAW \
    --cap-add NET_BIND_SERVICE \
    -e SLOGR_API_KEY=sk_free_XXXXXXXXXX \
    -e SLOGR_OTLP_ENDPOINT=https://otel.slogr.io/v1/metrics \
    -v /opt/slogr/data:/opt/slogr/data \
    slogr-agent:latest daemon

Run (CONNECTED — with sk_live_ key):
  docker run -d \
    --name slogr-agent \
    --cap-add NET_RAW \
    --cap-add NET_BIND_SERVICE \
    -e SLOGR_API_KEY=sk_live_XXXXXXXXXX \
    -e SLOGR_OTLP_ENDPOINT=https://otel.slogr.io/v1/metrics \
    -v /opt/slogr/data:/opt/slogr/data \
    slogr-agent:latest daemon

View logs:
  docker logs -f slogr-agent

Stop:
  docker stop slogr-agent


---  OPTION 2: SYSTEMD (Linux VMs / bare metal)  ---

Requirements:
  - Java 21 (Amazon Corretto recommended)
  - libslogr-native.so placed in /opt/slogr/lib/

1. Create user and directories:
     sudo useradd -r -s /sbin/nologin -d /opt/slogr slogr
     sudo mkdir -p /opt/slogr/lib /opt/slogr/data /opt/slogr/logs /etc/slogr
     sudo chown -R slogr:slogr /opt/slogr

2. Copy files:
     sudo cp slogr-agent-all.jar         /opt/slogr/
     sudo cp libslogr-native.so          /opt/slogr/lib/
     sudo cp deploy/wrapper.sh           /usr/local/bin/slogr-agent
     sudo cp deploy/slogr-agent.service  /etc/systemd/system/
     sudo chmod +x /usr/local/bin/slogr-agent
     sudo touch /etc/slogr/env && sudo chmod 600 /etc/slogr/env

3. Set API key (optional):
     echo 'SLOGR_API_KEY=sk_live_XXXXXXXXXX' | sudo tee /etc/slogr/env
     echo 'SLOGR_OTLP_ENDPOINT=https://otel.slogr.io/v1/metrics' | sudo tee -a /etc/slogr/env

4. Enable and start:
     sudo systemctl daemon-reload
     sudo systemctl enable slogr-agent
     sudo systemctl start slogr-agent

5. Verify:
     sudo systemctl status slogr-agent
     sudo journalctl -u slogr-agent -f

================================================================================
  REQUIRED LINUX CAPABILITIES
================================================================================

The agent needs two Linux capabilities:

  CAP_NET_RAW          Required for TWAMP raw UDP sockets and ICMP traceroute
  CAP_NET_BIND_SERVICE Required to bind port 862 (< 1024) as non-root user

In Docker, pass:     --cap-add NET_RAW --cap-add NET_BIND_SERVICE

In systemd service:  AmbientCapabilities=CAP_NET_RAW CAP_NET_BIND_SERVICE
                     CapabilityBoundingSet=CAP_NET_RAW CAP_NET_BIND_SERVICE

Without CAP_NET_RAW the TWAMP probe and traceroute will fail.
Without CAP_NET_BIND_SERVICE the reflector cannot bind port 862.

--------  slogr-agent check <target>  --------

Run a one-shot measurement against a target and print results.

  Usage:  slogr-agent check <host-or-ip> [options]

  Options:
    --port, -p <port>       TWAMP target port (default: 862)
    --profile <name>        SLA profile to evaluate against (default: internet)
    --format, -f <format>   Output format: text or json (default: text)
    --traceroute            Include traceroute in measurement

  Measurement flow:
    1. Attempts TWAMP (RFC 5357) to target:port
    2. If no TWAMP reflector: falls back to ICMP ping (5 probes)
    3. Attempts TCP connect on port 443 (then 80)
    4. Exports result to OTLP if SLOGR_OTLP_ENDPOINT is set

  Exit codes:
    0   Measurement succeeded (GREEN/YELLOW grade)
    3   Target unreachable (TWAMP, ICMP, and TCP all failed)
    1   Other error

  JSON output fields:
    measurement_method    "twamp" or "icmp"
    target                IP address
    profile               Profile name used
    grade                 GREEN, YELLOW, or RED
    twamp {}
      packets_sent        Integer
      packets_recv        Integer
      fwd_loss_pct        Float (%)
      fwd_min_rtt_ms      Float
      fwd_avg_rtt_ms      Float
      fwd_max_rtt_ms      Float
      fwd_jitter_ms       Float
      rev_min_rtt_ms      Float (optional — reflector timestamps)
      rev_avg_rtt_ms      Float (optional)
      rev_max_rtt_ms      Float (optional)
      rev_jitter_ms       Float (optional)
      session_id          UUID
      path_id             UUID
    ping {}               (present if TWAMP unavailable)
      sent                Integer
      received            Integer
      loss_pct            Float
      min_rtt_ms          Float (optional)
      avg_rtt_ms          Float (optional)
      max_rtt_ms          Float (optional)
    tcp {}                (present if TCP fallback attempted)
      port                Integer
      connect_ms          Float
    traceroute {}         (present if --traceroute used)
      direction           "uplink"
      hops []
        ttl               Integer
        ip                String
        rtt_ms            Float
        loss_pct          Float
        asn               Integer (if ASN DB configured)
        asn_name          String  (if ASN DB configured)

  Examples:
    slogr-agent check 8.8.8.8
    slogr-agent check 10.0.0.5 --port 862 --profile voip
    slogr-agent check 10.0.0.5 --format json
    slogr-agent check 10.0.0.5 --traceroute


--------  slogr-agent connect  --------

Register this agent with the Slogr backend using a live API key.
Only needed for CONNECTED mode (sk_live_* keys).
For sk_free_* keys, just set SLOGR_API_KEY and run daemon directly.

  Usage:  slogr-agent connect [--api-key <key>]

  Options:
    --api-key <key>   Live API key (sk_live_*). Prompts if not provided.

  What it does:
    1. Checks not already connected (error if already registered)
    2. Validates key format (must start with sk_live_)
    3. POSTs to SLOGR_API_URL/v1/agents with machine metadata:
         hostname, MAC address, IP addresses, cloud provider/region,
         OS info, Java version, CPU cores, memory
    4. Receives: JWT, RabbitMQ credentials, Pub/Sub subscription
    5. Encrypts and stores credentials to {SLOGR_DATA_DIR}/credential.enc

  Exit codes:
    0   Registered successfully
    1   Network error or API returned 4xx/5xx
    4   Invalid key format

  Error messages:
    "Already connected as <name>. Run 'slogr-agent disconnect' first."
    "Key must start with 'sk_live_'. For sk_free_* keys, set SLOGR_API_KEY."

  After connect, run:  slogr-agent daemon


--------  slogr-agent disconnect  --------

Remove stored credentials. Agent returns to ANONYMOUS or REGISTERED mode.

  Usage:  slogr-agent disconnect

  What it does:
    - Deletes {SLOGR_DATA_DIR}/credential.enc
    - Does NOT call a deregister API (agent stays in cloud as inactive)
    - Running daemon continues in disconnected mode
    - Agent can be re-registered with 'slogr-agent connect' later

  Exit code: 0 (always succeeds, even if not connected)


--------  slogr-agent daemon  --------

Run the agent as a background daemon. This is the primary production command.

  Usage:  slogr-agent daemon [--config <path>]

  Options:
    --config <path>   Path to schedule config JSON file.
                      If not specified, loads persisted schedule from
                      {SLOGR_DATA_DIR}/schedule.json

  Startup behavior (state-aware):
    CONNECTED + no stored credential  →  Auto-registers with api.slogr.io
    CONNECTED + stored credential     →  Connects directly via stored JWT
    REGISTERED                        →  OTLP export enabled, no queue
    ANONYMOUS                         →  stdout only, nudges for API key

  What runs in daemon mode:
    1. TWAMP reflector binds 0.0.0.0:862 (accepts inbound measurements)
    2. Loads schedule (--config file, or persisted, or empty)
    3. If no sessions: runs in responder-only mode (still accepts connections)
    4. Runs up to 20 concurrent measurement sessions on schedule cadence
    5. Records each result to OTLP exporter
    6. On SIGTERM: stops scheduler, flushes metrics, shuts down cleanly

  Signals:
    SIGTERM   Graceful shutdown (used by systemd stop / docker stop)


--------  slogr-agent setup-asn  --------

Configure MaxMind GeoLite2-ASN database for ASN enrichment in traceroutes.
Without it, traceroute hops show IP addresses only (no AS numbers or names).

  Usage:  slogr-agent setup-asn [--db-path <path>]

  Options:
    --db-path <path>   Path to GeoLite2-ASN.mmdb
                       Default: {SLOGR_DATA_DIR}/GeoLite2-ASN.mmdb

  To get the database (free):
    1. Register at https://www.maxmind.com/en/geolite2/signup
    2. Download GeoLite2-ASN.mmdb
    3. Place at the path above
    4. Run 'slogr-agent setup-asn' to verify

Run diagnostics to verify the agent environment is correctly configured.

================================================================================
  CREDENTIAL STORAGE
================================================================================

Credentials are stored encrypted at:  {SLOGR_DATA_DIR}/credential.enc

Encryption:
  - AES-256-GCM
  - Key derived from machine fingerprint (SHA-256 of hostname + MAC)
  - File format: [12-byte IV][AES-GCM ciphertext]
  - File permissions: 0600 (owner read/write only)

The credential file contains:
  - agentId, tenantId, displayName
  - JWT for API calls
  - RabbitMQ host, port, JWT
  - GCP Pub/Sub subscription path
  - Connection timestamp and method

For sk_free_* keys, a validation cache is stored at:
  {SLOGR_DATA_DIR}/.slogr-key-cache.json
  TTL: 24 hours. Expired entries trigger re-validation via API.

        "jitter_red_ms":    30.0,

        "loss_red_pct":     1.0

      "interval_seconds":   300,
      "traceroute_enabled": true,
      "skip_cycles":        0,

================================================================================
  DATA FLOW OVERVIEW
================================================================================

For inbound connections (reflector mode):
  Remote agent connects to 0.0.0.0:862 (TCP)
    → TwampReflector accepts and runs state machine
    → Negotiates TWAMP session, opens UDP reflector port
    → Reflects test packets back with timestamps

For outbound measurements (controller mode):
  TestScheduler fires on interval
    → TwampController connects to target:862 (TCP)
    → Exchanges TWAMP control messages
    → Sends/receives UDP test packets
    → MeasurementAssembler builds MeasurementResult
    → SlaEvaluator assigns grade
    → OtlpExporter records metrics
    → onResult callback logs grade

================================================================================
  KNOWN REQUIREMENTS AND CONSTRAINTS
================================================================================

- Java 21 required (Amazon Corretto 21 recommended)
- Linux required for production use (JNI + raw sockets)
- JVM heap: 384 MB default, total process ceiling ~512 MB (suitable for t3.micro)
- Port 862/TCP must be reachable between agents for TWAMP
- CAP_NET_RAW required for TWAMP UDP and ICMP traceroute
- CAP_NET_BIND_SERVICE required to bind port 862 as non-root
- MaxMind GeoLite2-ASN.mmdb needed for ASN enrichment (optional)
- OTLP endpoint must be HTTP (not gRPC) — agent uses OTLP/HTTP exporter

Windows note: The agent can run on Windows in development mode using Java
  DatagramSocket fallbacks (JavaUdpTransport), but JNI features (TWAMP raw
  UDP, ICMP traceroute) require Linux. All production deployments are Linux.
